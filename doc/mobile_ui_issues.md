# Проблемы мобильного UI в QubeForge (Ландшафтный режим)

В ходе тестирования игры в режиме мобильного ландшафта (например, 800x400 или 667x375) были выявлены критические проблемы с доступностью элементов управления в меню.

## Общие системные проблемы

1. **Вертикальное переполнение без прокрутки**: 
   Большинство меню центрируются по вертикали с помощью `flexbox` на весь экран. Если высота контейнера меню больше высоты экрана, верхняя (заголовки) и нижняя (кнопки навигации) части выходят за пределы видимости и становятся недоступными.

2. **Отсутствие адаптации под малую высоту**: 
   Медиа-запросы в CSS в основном ориентированы на ширину (`max-width: 600px`), но не учитывают сценарии с малой высотой экрана (`max-height`).

## Конкретные окна и меню

### 1. Меню настроек (Settings Menu) [FIXED]
- **Проблема**: Кнопка "Назад" (Back) находится ниже границы экрана.
- **Результат**: Пользователь не может выйти из настроек обратно в главное меню или меню паузы.

### 2. Подменю "Звук" и "Управление" [FIXED]
- **Проблема**: Кнопки "Сбросить" и "Назад" находятся за пределами экрана.
- **Результат**: Невозможно сохранить изменения или вернуться назад. В меню управления список клавиш имеет прокрутку, но сами кнопки управления меню находятся вне этого списка.

### 3. Диалог создания мира (New World Dialog) [FIXED]
- **Проблема**: Кнопки "Создать" и "Отмена" полностью отрезаны внизу.
- **Результат**: Невозможно создать новый мир в ландшафтном режиме.

### 4. Выбор мира (World Selection) [FIXED]
- **Проблема**: Список миров занимает слишком много места, прижимая кнопки "Создать Мир" и "Назад" к самому краю экрана.
- **Результат**: Сложность взаимодействия на устройствах с системными жестами (навигационные бары Android/iOS).

### 5. Игровой HUD (Heads-Up Display) [FIXED]
- **Проблема**: Кнопка прыжка ("ПРЫГ") перекрывает правую часть хотбара.
- **Проблема**: Полоска здоровья и хотбар расположены слишком глубоко в центре экрана по вертикали, оставляя пустое место снизу.
- **Результат**: Ограничение видимости игрового мира и случайные нажатия на элементы HUD вместо управления.

### 6. Инвентарь в игре (In-game Inventory) [FIXED]
- **Проблема**: Мобильные элементы управления (джойстик и кнопки действий) не скрываются при открытии инвентаря.
- **Проблема**: Окно инвентаря слишком высокое для ландшафтного режима и обрезается снизу.
- **Результат**: Наложение кнопок управления на слоты инвентаря делает невозможным нормальное перемещение предметов.

### 7. Интерфейс Печи (Furnace UI) [FIXED]
- **Проблема**: Интерфейс печи добавляется внутрь контейнера инвентаря, увеличивая его общую высоту.
- **Проблема**: В ландшафтном режиме суммарная высота (Печь + Инвентарь) значительно превышает высоту экрана (например, ~580px против 400px).
- **Результат**: Верхняя часть интерфейса (слот входа) или нижняя часть инвентаря уходят за пределы экрана и становятся недоступными.

### 8. Главное меню [FIXED]
- **Проблема**: Кнопки расположены слишком близко к правому краю, а логотип занимает много места по вертикали.

## Рекомендации по исправлению

1. **Добавить `overflow-y: auto` и `max-height`**: 
   Для всех контейнеров меню (`#settings-menu-container`, `.creation-form` и т.д.) установить `max-height: 90vh` или `max-height: calc(100% - 20px)`.
   
2. **Использовать медиа-запросы по высоте**:
   ```css
   @media (max-height: 500px) {
       .menu-btn {
           padding: 10px 15px;
           margin-bottom: 10px;
       }
       .setting-item {
           margin: 10px;
       }
       h2 {
           margin-bottom: 10px;
       }
   }
   ```

3. **Адаптация компоновки**: 
   В ландшафтном режиме рассмотреть возможность разделения меню на две колонки (например, настройки слева, кнопки справа) или уменьшение вертикальных отступов.
